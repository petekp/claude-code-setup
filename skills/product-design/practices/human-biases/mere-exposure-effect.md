## Principle
Users develop preference for things they encounter repeatedly — familiarity breeds liking, not contempt, which means consistent exposure to interface patterns increases user comfort and satisfaction over time.

## Why It Matters
Users initially resist new interface patterns not because the patterns are bad but because they are unfamiliar. Given repeated exposure, the same patterns that felt confusing become comfortable and eventually preferred. This explains why users complain loudly about redesigns but gradually come to prefer the new design. It also explains why design consistency across products in an ecosystem (Material Design, Apple HIG) works — cross-product familiarity transfers positive associations. The mere exposure effect means that persistence and consistency are legitimate design strategies, not just stubbornness.

## Application Guidelines
- Maintain visual and interaction consistency throughout the product so that familiarity builds with use
- When introducing new patterns, provide repeated exposure in low-stakes contexts before using them for critical actions
- Leverage established platform conventions (iOS, Android, web standards) because users already have familiarity and positive associations
- In redesigns, introduce changes gradually rather than all at once to allow familiarity to build incrementally
- For features you want users to adopt, increase their visibility across multiple touchpoints rather than relying on a single announcement

## Anti-Patterns
- Redesigning the entire interface simultaneously, forcing users to build familiarity from scratch with every element
- Using inconsistent patterns within the same product, preventing familiarity from accumulating
- Inventing novel interaction patterns when standard ones exist, requiring users to overcome initial unfamiliarity unnecessarily
- Hiding new features behind a single menu item instead of exposing them in contextually relevant locations where repeated exposure can build comfort

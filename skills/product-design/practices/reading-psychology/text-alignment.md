## Principle
Left-align body text for optimal readability; reserve center alignment for short headings and avoid justified text in digital interfaces entirely.

## Why It Matters
Left-aligned text creates a consistent left edge (called a "hard rag") that the eye uses as an anchor point when returning to the start of each new line. Justified text creates uneven spacing between words (called "rivers") that disrupts reading flow and is especially problematic on narrow screens where gaps become visually distracting. Centered body text forces the eye to hunt for the start of each line, since the left edge shifts unpredictably — this is manageable for 2-3 short lines but painful for paragraphs.

## Application Guidelines
- Left-align all body text, form labels, list items, and any content longer than two lines
- Center-align only short, display-level text: headings, hero text, card titles, or single-line captions
- Never justify text on the web — unlike print typesetting, browsers lack sophisticated hyphenation and kerning algorithms, producing visually uneven word spacing
- Right-align only numeric data in tables (so decimal points align) and certain UI metadata like timestamps
- For RTL languages, mirror the principle: right-align body text to create the consistent reading edge

## Anti-Patterns
- Center-aligning paragraph text in marketing pages for aesthetic symmetry, creating a ragged left edge that forces users to re-find the start of each line, dramatically slowing reading speed

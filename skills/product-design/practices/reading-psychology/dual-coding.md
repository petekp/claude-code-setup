## Principle
When text is accompanied by a relevant visual — an icon, illustration, diagram, or image — comprehension and recall improve because the brain encodes the information through two independent channels.

## Why It Matters
Allan Paivio's dual coding theory demonstrates that verbal and visual information are processed by separate cognitive systems. When both systems encode the same concept simultaneously, the resulting memory trace is stronger and more retrievable than either channel alone. In interface design, this means that a status message paired with a matching icon, or a feature description paired with a relevant illustration, is understood faster and remembered longer. The critical requirement is semantic alignment — the visual must reinforce the text, not merely decorate the page.

## Application Guidelines
- Pair status messages with semantically matched icons: a checkmark for success, a warning triangle for alerts, an X for errors
- Use diagrams and flowcharts to supplement process descriptions — spatial representation of sequences outperforms text-only instructions
- Add relevant screenshots or illustrations to help documentation; users comprehend "click the blue button in the top right" faster when they can see it
- Use icons as scanning aids in navigation and lists, but always include text labels — icons without labels are ambiguous
- Ensure visual-text pairs are spatially close (Gestalt proximity) so the brain processes them as a unit rather than separate elements

## Anti-Patterns
- Adding decorative stock illustrations that have no semantic relationship to the adjacent text — this consumes visual attention without reinforcing comprehension and can actively confuse users who try to interpret the image as meaningful

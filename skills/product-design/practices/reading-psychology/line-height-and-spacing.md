## Principle
Set line height (leading) to 1.4-1.6x the font size for body text, and maintain a consistent vertical rhythm throughout the page to reduce reading fatigue and improve comprehension.

## Why It Matters
Line height controls the vertical space between lines of text. When it is too tight, lines visually merge and the eye struggles to track from the end of one line to the start of the next — a problem called "doubling" where readers accidentally re-read the same line. When it is too loose, the visual connection between lines weakens and the text feels fragmented. The 1.4-1.6x range optimizes the balance between density and readability. Consistent vertical rhythm — where all spacing follows a baseline grid — creates a visual cadence that the eye finds restful and predictable.

## Application Guidelines
- Set body text line height to 1.5 (150%) as a default; adjust to 1.4 for larger text and 1.6 for smaller text
- Increase line height for wider line lengths — long lines need more vertical space so the eye can track back accurately
- Use a baseline grid (typically 4px or 8px) and ensure all vertical spacing (line height, margins, padding) snaps to this grid
- Add extra line height for text on colored or dark backgrounds, where reduced contrast makes tracking harder
- Reduce line height slightly (1.1-1.2) for headings and display text, which are read as short phrases rather than continuous prose

## Anti-Patterns
- Using the browser default line height of 1.2 (120%) for body text, which is too tight for comfortable reading of paragraphs and causes line-tracking errors, especially on screens wider than mobile

## Principle
Form fields should be chunked into small, semantically meaningful groups using Proximity — tight spacing within a chunk and generous spacing between chunks — so users process forms in manageable pieces rather than as an undifferentiated list.

## Why It Matters
Long forms are a primary cause of user abandonment. Chunking leverages Proximity to break a daunting wall of fields into digestible clusters (e.g., "Personal Info," "Address," "Payment"), which reduces perceived complexity and gives users a sense of progress as they complete each chunk. This is especially valuable for users with ADHD or working memory limitations who benefit from clear structural boundaries.

## Application Guidelines
- Group 2-5 related fields into a chunk with 8-12px vertical spacing between fields and 24-40px spacing (or a visual divider) between chunks
- Place inline field groups (e.g., first name + last name, city + state + zip) on the same row with tighter horizontal spacing to signal they are sub-parts of one concept
- Use a visible section label or subtle background shift at each chunk boundary to reinforce the Proximity grouping with a secondary cue
- When designing for mobile where vertical space is at a premium, maintain the spacing ratio between chunks even if absolute values shrink — the ratio is what creates the grouping, not the absolute gap

## Anti-Patterns
- Using uniform spacing between every field so the form reads as one long, undifferentiated list
- Creating chunks of 8-10 fields, which are too large to function as cognitive units and defeat the purpose of chunking
- Separating fields that users think of as a single concept (e.g., putting "Street Address" and "City" in different chunks) because the grouping does not match the user's mental model

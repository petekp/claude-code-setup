## Principle
Every view must establish an unambiguous focal point so the user's eye instantly separates the primary content (figure) from the surrounding context (ground).

## Why It Matters
When figure and ground are ambiguous, users stall — they cannot determine where to look first or what action to take. This increases time-to-task, raises cognitive load, and causes particular difficulty for users with low vision or attention-related conditions who depend on strong visual hierarchy to orient themselves.

## Application Guidelines
- Use contrast (color, size, or weight) to make exactly one element per view the obvious focal point; if everything is bold, nothing is bold
- Apply depth cues such as drop shadows, elevation, or background dimming to lift the primary content plane away from secondary content
- Limit the number of competing visual weights on a single screen — audit for "visual shouting" where multiple elements fight for attention
- When presenting data visualizations, ensure the data layer (figure) has significantly higher contrast than axes, gridlines, and labels (ground)

## Anti-Patterns
- Giving every card, button, and heading the same visual weight so the user has no entry point
- Using ambiguous overlapping layers where it is unclear which element is in front and which is behind
- Placing high-contrast decorative elements (illustrations, background patterns) that compete with the actual content for figure status

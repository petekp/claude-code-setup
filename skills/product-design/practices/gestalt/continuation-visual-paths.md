## Principle
The eye naturally follows lines, curves, and aligned edges, so designers should arrange elements along continuous visual paths to guide attention in the intended sequence.

## Why It Matters
Users do not read interfaces randomly; they follow visual momentum. When elements align along a clear axis or curve, the eye moves fluidly from one to the next. Broken alignment or jarring directional changes interrupt this flow, forcing users to visually "restart" and increasing the chance they miss important content.

## Application Guidelines
- Align related content along a shared vertical or horizontal axis so the eye travels a single smooth path through the group
- Use leading lines, directional cues (arrows, chevrons), or gradient fades to indicate continuation beyond the visible viewport
- In multi-step flows (wizards, onboarding), create a visible path (stepper bar, progress line) that the eye can follow from start to finish
- Maintain consistent left-edge alignment in lists and forms so the eye does not zigzag between indentation levels

## Anti-Patterns
- Centering some form labels and left-aligning others within the same form, breaking the continuous reading path
- Placing sequential steps in a non-linear spatial arrangement (e.g., zigzagging cards) without visual connectors
- Using inconsistent margins that misalign content across sections, fragmenting the vertical reading flow

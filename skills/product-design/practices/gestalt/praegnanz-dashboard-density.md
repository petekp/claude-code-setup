## Principle
Dashboard layouts should achieve the simplest possible visual organization for the amount of data presented, using regular grids and uniform modules to let users perceive order before processing content.

## Why It Matters
Dashboards are inherently dense — they exist to surface many data points at once. The Law of Praegnanz means users will try to perceive the simplest structure. If the layout is chaotic (mixed card sizes, misaligned edges, irregular gutters), the brain spends effort just parsing the layout before it can begin parsing the data. A regular, predictable grid lets cognitive resources go directly to the information.

## Application Guidelines
- Use a strict column grid with consistent gutters so every card edge aligns with its neighbors, creating the simplest possible spatial pattern
- Standardize card sizes to as few variants as possible (e.g., 1x1, 2x1, 2x2) so the dashboard reads as a regular arrangement of uniform modules
- When information density must increase, add data within existing cards (e.g., tabs, expandable sections) rather than cramming more irregularly sized cards into the layout
- Leave intentional breathing room — Praegnanz does not mean "pack everything in," it means the arrangement should feel ordered and inevitable

## Anti-Patterns
- Allowing every team to define their own card dimensions, resulting in a patchwork layout with no discernible grid
- Breaking alignment between cards so edges do not line up, forcing the eye to process each card's position independently
- Filling every pixel of a dashboard with data, eliminating the whitespace that makes the grid structure perceivable

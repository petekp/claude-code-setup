## Principle
People remember and are drawn to incomplete tasks more than completed ones — strategically surfacing unfinished work creates a psychological pull that drives users to return and re-engage.

## Why It Matters
The Zeigarnik effect, first documented by Bluma Zeigarnik in 1927, shows that interrupted or incomplete tasks create a state of cognitive tension that persists until the task is resolved. The human brain naturally seeks closure, and unfinished items occupy mental bandwidth even when the user is away from the product. In product design, this means that visible incomplete states — unfinished profiles, partially completed projects, abandoned workflows — are not just status indicators; they are motivational hooks that pull users back to resolve the open loop. Used ethically, this effect improves onboarding and task completion. Used exploitatively, it creates anxiety and compulsive engagement patterns. The line between helpful motivation and manipulative pressure matters.

## Application Guidelines
- Show "profile completeness" indicators that highlight specific missing elements, creating a visible gap the user is motivated to close
- Send well-timed reminders about in-progress work: "You have a draft report with 3 sections remaining"
- Design save states that make returning to incomplete work frictionless — one click to resume exactly where the user left off
- Use progress visualization (completion rings, checklists with some items checked) to make the gap between current and complete state visually salient
- Create natural breaking points in long workflows that leave users mid-task but not mid-thought, so the Zeigarnik pull is strong without the frustration of interruption
- Use checklists in onboarding and setup flows to create visible open loops that naturally motivate completion

## Anti-Patterns
- Manufacturing artificial incompleteness (endless progress bars that never reach 100%, gamified checklists that always have "one more thing") that teaches users the completion target is unreachable, destroying the motivational tension the Zeigarnik effect depends on
- Creating artificial "incomplete" states to manipulate return visits (e.g., "Your profile is only 60% complete" when the missing fields are unnecessary)
- Sending aggressive reminders about abandoned processes that create anxiety rather than helpfulness
- Showing incomplete task counts as badges that create guilt rather than motivation

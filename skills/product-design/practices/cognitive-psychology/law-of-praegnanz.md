## Principle
The human visual system interprets ambiguous or complex shapes as the simplest possible form, so interfaces should use the simplest visual representation that accurately communicates meaning.

## Why It Matters
Visual complexity has a direct cognitive cost. Every extra border, shadow, gradient, decorative element, and visual flourish requires processing power that could be spent understanding content. The Law of Praegnanz (also called the Law of Good Figure or Law of Simplicity) tells us that perception naturally seeks order and simplicity — interfaces that align with this tendency feel clean and comprehensible, while those that fight it feel cluttered and overwhelming. Simplicity is not a stylistic preference; it is a cognitive performance optimization.

## Application Guidelines
- Use the minimum number of visual properties needed to communicate each element's function — if a border alone distinguishes a card, don't add a shadow, background color, and border simultaneously
- Prefer simple geometric shapes (rectangles, circles) for UI elements; avoid decorative shapes that add visual complexity without functional meaning
- Reduce icon complexity: use consistent-weight outline or filled icons with clear silhouettes, avoiding detailed illustrative icons for functional UI
- Limit the color palette to a purposeful set where each color carries semantic meaning rather than decorative variety
- Favor flat or subtly elevated design over heavy skeuomorphism — depth cues should be functional (indicating interactivity or layering), not decorative
- When presenting data, choose the simplest chart type that accurately represents the data relationship (e.g., a bar chart over a 3D bar chart)

## Anti-Patterns
- Applying drop shadows, borders, gradients, and background colors simultaneously to a single card element when any one treatment would suffice
- Using complex illustrative icons in the navigation bar that take longer to parse than simple line icons
- A chart with 3D perspective, grid lines, gradient fills, and decorative elements when a simple 2D chart with minimal gridlines would communicate the same data more clearly
- Adding visual embellishments (decorative dividers, ornamental shapes, texture overlays) that increase visual complexity without improving comprehension

## Principle
Work expands to fill the time and space available — constrain form scope, field sizes, and workflow steps to prevent unnecessary task inflation.

## Why It Matters
When given an unconstrained text area, users either write far more than needed (inflating review burden and processing time) or feel overwhelmed by the open-endedness and abandon the field entirely. Similarly, when workflows have no deadlines or step limits, they expand indefinitely. Thoughtful constraints channel effort toward what matters and signal to users that the system respects their time by not demanding more than necessary.

## Application Guidelines
- Set explicit character limits on text fields and display a remaining character count to encourage concise input
- Use structured inputs (dropdowns, rating scales, multiple choice) instead of open-ended text fields when the needed information can be categorized
- Break sprawling workflows into time-boxed phases with clear deliverables at each stage
- For description or comment fields, provide prompt text that models the expected length and specificity (e.g., "In 1-2 sentences, describe...")
- Set deadlines and reminders for multi-day workflows to prevent indefinite expansion
- Limit the number of items users can select or create when unlimited selection would reduce quality (e.g., "Choose your top 3 priorities" rather than "Select all that apply")

## Anti-Patterns
- Providing a full-page, unconstrained rich text editor for a field that only needs a one-sentence summary, resulting in users writing 500-word essays that no one reads and that slow down every downstream process

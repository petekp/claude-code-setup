## Principle
Let users experience the core value of a product before requiring them to create an account, enter payment information, or make any commitment — front-load the value, back-load the friction.

## Why It Matters
Requiring signup before demonstrating value is asking users to invest before they know if the return is worthwhile. This is the primary reason landing pages and signup forms have high abandonment rates. Gradual engagement flips this: let users try the tool, see results, start building something — and then ask them to create an account to save their work. By the time the friction point arrives, the user has already experienced value and has a concrete reason to continue. This pattern dramatically increases conversion because the commitment decision is informed rather than speculative.

## Application Guidelines
- Allow users to start the core task without creating an account: edit a document, run a calculation, explore a dataset, generate a result
- Gate features that genuinely require identity (saving work, collaboration, account settings) rather than the initial experience
- When asking for account creation, remind users of the value they have already created: "Create an account to save your project with 12 items"
- Use progressive account creation: start with just an email, add a password later, collect profile details even later — each step justified by a new capability
- For free trials, skip the credit card requirement at signup and only request it when the trial is ending and the user has demonstrated engagement
- Design the first-run experience to deliver an "aha moment" as quickly as possible — the sooner users see value, the more willing they are to commit

## Anti-Patterns
- Requiring a 10-field registration form with email verification before a user can see any functionality of the product, forcing them to commit time and personal information on faith that the product will be worth it

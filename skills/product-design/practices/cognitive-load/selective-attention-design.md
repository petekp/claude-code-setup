## Principle
Each screen or view should have exactly one primary focal point that immediately communicates where the user should direct their attention.

## Why It Matters
Human attention is a finite, serial resource. When a screen presents multiple competing visual elements of equal weight, users must consciously decide where to look first, consuming cognitive resources before they even engage with content. A single clear focal point leverages the brain's automatic orientation response, reducing the decision overhead of "where do I start?" to zero. This is especially critical for first-time users and high-stakes tasks like checkout flows.

## Application Guidelines
- Establish visual hierarchy through size, contrast, color, and whitespace so the primary element is unambiguously dominant
- Limit each screen to one primary call-to-action; secondary actions should be visually subordinate
- Use the F-pattern or Z-pattern as a baseline for content placement, putting the focal point where eyes naturally land first
- Dim, blur, or reduce the prominence of non-essential elements during focused tasks (e.g., modal overlays)
- Test focal points with 5-second usability tests — if users can't identify the primary action within 5 seconds, the hierarchy is broken

## Anti-Patterns
- Multiple elements competing for attention through equal sizing, bold colors, or animation
- "Christmas tree" dashboards where every metric is highlighted as equally important
- Placing the primary action below the fold or in a visually recessive position
- Using attention-grabbing design (bright colors, animation) on non-essential elements

## Principle
Provide persistent, visible external representations of information so users can offload memory and tracking tasks to the interface itself.

## Why It Matters
Humans naturally offload cognitive work to their environment — writing notes, using bookmarks, arranging physical objects as reminders. Digital interfaces should serve the same function. When an interface provides visible state indicators, persistent histories, and accessible reference information, users can redirect the mental energy they would spend on remembering and tracking toward higher-value thinking like analysis, decision-making, and creative work.

## Application Guidelines
- Show persistent indicators for all active states (filters applied, sort order, selected items, unsaved changes)
- Provide search history, recent items, and "last visited" lists so users don't need to remember paths
- Display running totals, character counts, and progress metrics in real-time as users work
- Offer comparison views, pinned items, and clipboards that let users park information for later reference
- Make undo/redo history visible and accessible so users can offload the burden of "remembering what I did"

## Anti-Patterns
- Clearing search fields after navigation, forcing users to re-type queries
- Resetting filter and sort states when users return to a list view
- Not showing which filters or criteria are actively applied to a dataset
- Requiring users to remember item IDs, reference numbers, or configuration values across screens

## Principle
When users need to learn, invest their cognitive resources in schema-building activities (understanding patterns, making connections) rather than in decoding the interface itself.

## Why It Matters
Germane cognitive load is the "good" kind of cognitive work — the effort users spend building mental models, recognizing patterns, and integrating new knowledge with what they already know. In learning interfaces (onboarding flows, tutorials, documentation, educational products), the design goal is to maximize germane load while minimizing extraneous load. An interface that is itself confusing steals cognitive resources from learning; a clean, well-structured interface channels all available cognitive capacity into understanding the content.

## Application Guidelines
- Use worked examples that show the complete process before asking users to practice — this builds schemas more efficiently than trial-and-error
- Present new concepts with clear analogies to things users already understand, building on existing schemas
- Vary examples and practice scenarios to promote flexible schema formation rather than rote memorization
- Encourage self-explanation by prompting users to articulate why something works, not just how (e.g., "Why did this configuration produce this result?")
- Structure learning content so each new concept builds on the previously established one, creating a coherent knowledge chain

## Anti-Patterns
- Tutorial interfaces so complex that users spend more effort navigating the tutorial than learning the content
- Teaching through lengthy text explanations instead of interactive, schema-building activities
- Presenting all concepts at the same level without building from foundational to advanced
- Practice environments that punish exploration and errors rather than using them as learning opportunities

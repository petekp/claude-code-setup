## Principle
Remove all interface elements, decorations, and interactions that do not directly support the user's current task or learning goal.

## Why It Matters
Cognitive load theory distinguishes three types of load: intrinsic (inherent task complexity), germane (productive learning effort), and extraneous (unnecessary overhead introduced by poor design). Extraneous load — decorative graphics, redundant text, confusing layouts, unnecessary steps — steals cognitive capacity from the intrinsic and germane processing that users actually need. Every pixel, word, and interaction that doesn't earn its place on the screen is actively working against the user.

## Application Guidelines
- Audit every element on screen by asking "does this help the user complete their current task?" — if not, remove or hide it
- Replace decorative imagery with functional visuals (diagrams, data visualizations, instructional illustrations)
- Eliminate redundant copy: don't say the same thing in a heading, subheading, and body paragraph
- Simplify interaction sequences by removing unnecessary confirmation dialogs, intermediate screens, and extra clicks
- Use whitespace strategically to create visual breathing room, not just to fill space

## Anti-Patterns
- Decorative stock photos that add visual weight without communicating information
- Marketing banners and promotional content injected into task-focused interfaces
- Mandatory tutorials or walkthroughs for straightforward interfaces
- Multiple navigation systems (sidebar, top bar, breadcrumbs, footer links) all competing for attention

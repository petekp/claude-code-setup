## Principle
Make the immediate benefit of each action visible and tangible so the cognitive cost of engagement feels worthwhile in the moment.

## Why It Matters
Temporal discounting — the human tendency to heavily favor immediate rewards over future ones — means that interfaces requiring upfront effort for deferred benefits face an inherent engagement problem. Users will abandon a lengthy onboarding flow even if completing it would save them hours later. They'll skip profile setup, ignore configuration options, and avoid reading documentation — not because they're lazy, but because their brains systematically undervalue future payoffs. Aligning the interface so that each step delivers immediate, visible value transforms compliance from a willpower problem into a natural behavior.

## Application Guidelines
- Show immediate results at each step of onboarding (e.g., display the dashboard with real data after the first integration, not after all integrations)
- Provide instant previews as users make configuration choices so they see the impact in real time
- Frame feature discovery as solving the user's current problem, not as learning for later
- Use "quick wins" in early interactions to build a sense of progress and momentum
- When deferred value is unavoidable, make the future benefit concrete and specific ("This will save you ~2 hours per week on report generation")

## Anti-Patterns
- Long setup processes that deliver no visible value until the very end
- Abstract promises of future productivity without concrete evidence ("This will help you later!")
- Requiring extensive data entry before showing any useful output
- Tutorials that teach features in the abstract rather than in the context of a real task
